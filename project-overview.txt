This project is an AI website widget platform. A business signs in, creates one or more chat widgets, adds the business context/instructions for each widget, customizes the widget look, restricts where it can run by domain, and then installs it on a website with a single script tag. The backend also serves the embeddable widget itself, not just the API.

What exists in the product today, based on the code:

User authentication with Google sign-in and secure session cookies.
A private dashboard for managing widgets.
Widget creation, editing, listing, viewing, and deletion.
Per-widget business context, which is the knowledge/information the assistant answers from.
Optional per-widget system instructions to control assistant behavior/tone.
Domain allowlisting, so a widget can be limited to specific websites.
Widget appearance customization with at least primary color and panel background color.
A public embeddable chat widget that opens from the bottom-right corner of a website.
Widget UI with launcher, welcome message, title, placeholder text, send state, error state, and responsive mobile behavior.
A public chat endpoint that replies using the widget’s saved context.
AI processing through OpenRouter.
Two AI usage modes:
Platform-managed AI credentials.
User-provided OpenRouter credentials stored at account level.
Optional custom model selection when using user credentials.
Account settings for profile info, security/session info, AI credentials, installation info, and destructive actions.
Installation helpers, including generated embed snippets, script URL, API base URL, and demo URL.
Dashboard summary data such as total widgets, restricted vs public widgets, recent widgets, and quick embed snippet.
Logout current session, logout all sessions, delete all widgets, and delete account.
Rate limiting, encrypted AI credentials, structured errors, and logging on the backend.
A frontend developer should understand the product as: a SaaS-style tool for businesses to add an AI support/sales/help assistant to their website without building chat infrastructure themselves.

The main value proposition from the codebase is:

Create an AI assistant per website or use case.
Control what it knows through widget context.
Control where it appears through domain restrictions.
Control how it looks through basic theming.
Control how it is powered through platform AI or customer AI credentials.
Install it quickly with a lightweight embed script.
A few important scope notes from the current implementation:

Auth is Google-only right now.
AI provider is OpenRouter-only right now.
The widget is text chat only.
The knowledge source is manually stored widget context, not document crawling or file ingestion.
There is no visible billing, team management, analytics, or conversation history system in this backend.
